# 《创AI》申报总纲

> 本文档是当前仓库中《创AI》申报材料的总入口与口径总控。  
> 上游依据：`docs/source_materials/competition_guides/附件3：创AI案例征集指南.docx`

## 一、当前申报定位

- 案例名称：**我的计算森林：面向小学数学错因诊断与引导的教育智能体**
- 申报类别：**教育智能体**
- 申报学段：**小学**
- 当前主交付：**教师工作台 + 错因诊断闭环 + 教师审核边界**
- Dify 角色：**主交付入口与主演示链路**
- 森林相关设计：**保留品牌隐喻与温和成长语气，不作为当前 MVP 主叙事**

## 二、《创AI》指南的关键要求

### 1. 案例定义

《创AI》征集的是：教师在教育教学过程中借助生成式人工智能**自主编写、创新开发**各类智能教学工具和信息系统，并进行实践应用的案例。

对本项目最相关的定义是：

- **教育智能体**：基于国产生成式大模型、国产智能体平台或开发框架，通过提示词工程、知识库建设、工作流设计、多模型协同等方式构建具备特定教学功能与交互逻辑的智能体应用。

### 2. 必交材料

必须提交四类材料：

1. 案例信息表
2. 开发与应用报告
3. 演示视频
4. 配套资源 ZIP

### 3. 形式要求

- 案例信息表：Word + PDF，PDF 版需盖章
- 开发与应用报告：Word，**不超过 3000 字**
- 演示视频：MP4，**不超过 8 分钟**
- 配套资源：ZIP，需支持他人理解与复现

### 4. 评价重点

当前最应围绕的五项是：

- 导向性：价值导向正确，符合教育规律与人工智能伦理
- 实用性：基于真实问题，真正解决问题，且简单易用
- 影响力：有真实落地和分享推广价值
- 创新性：开发角度和应用模式有特色
- 完整性：代码、文档、流程和证据完整，便于复现

### 5. 合规边界

- 不得出现政治原则性错误、概念性错误和弄虚作假
- 需对原创性、真实性负责
- 不出现学生正面画面
- 如展示 AI 生成内容，需标记 **“AI生成”**
- 必须遵守《生成式人工智能服务管理暂行办法》

## 三、为什么本项目报“教育智能体”

本项目更适合“教育智能体”，而不是“智能信息系统”，原因如下：

1. 当前主交付不是班级数据中台或教务平台，而是围绕教师教学决策的智能交互工作流。
2. 系统核心能力来自提示词、知识库、工作流编排、规则诊断服务和教师审核边界的协同。
3. Dify 已作为国产智能体平台被指南明确认可，当前仓库中已有真实导入、运行与验证记录。
4. 项目的当前 MVP 更强调“教师如何与智能体协作完成诊断、引导和练习生成”，而不是“系统报表平台”。

## 四、为什么 Dify 必须保留

### 保留结论

**Dify 继续保留，而且应作为当前《创AI》材料中的主交付入口。**

### 保留理由

1. 指南明确允许使用国产智能体平台开展教育智能体开发。
2. 当前仓库已存在 Dify 工作流资产、本地导入记录、draft run 结果和配套文档，具备开发证据价值。
3. Dify 天然适合展示以下《创AI》加分项：
   - 提示词工程
   - 知识库建设
   - 工作流设计
   - 可复盘日志
   - 教学智能体的交互逻辑

### 必须约束的表述

- Dify 负责编排、交互与可视化
- FastAPI 规则服务负责结构化数学错因诊断
- 教师审核负责最终课堂使用边界
- 不宣称 Dify 独立完成全部产品能力
- 不宣称 LLM 负责算术正确性判断

## 五、当前提交策略

### 主演示链路

比赛材料默认采用以下主线：

```text
真实教学问题
  -> 教师在 Dify / 教师工作台输入题目与学生作答
  -> FastAPI 规则服务返回错因、证据、练习草案
  -> Dify 负责编排与呈现
  -> 教师审核后用于讲评和短时巩固
```

### 前端工作台的定位

前端工作台不是与 Dify 冲突的替代品，而是：

- 教师侧产品壳
- 当前真实应用展示界面
- 课堂模式与审核边界的补充演示面

### 森林相关内容的定位

当前对外口径统一为：

- 森林是品牌表达与成长语气层
- 不把“累计很多棵树”当作当前 MVP 主交付
- 不把学生长期成长系统写成已稳定落地功能

## 七、申报禁区清单

以下内容在比赛主材料中必须避免：

1. 把未实现能力写成已落地功能
2. 把学生端完整产品、家长端、长期森林系统写成当前主交付
3. 弱化或绕过教师审核边界
4. 暗示算术正确性由大模型判断
5. 用“可爱森林收集系统”替代真实教学问题
6. 只讲概念，不提供可复现的开发过程和应用证据

## 九、当前边界一句话总结

我们当前提交的不是“一个已经做完全部学生成长生态的产品”，而是一个**基于 Dify 与规则诊断服务构建的、面向小学数学教师的教育智能体**：它能帮助老师更快看清错因、组织短时反馈，并在教师审核边界内落地使用。
