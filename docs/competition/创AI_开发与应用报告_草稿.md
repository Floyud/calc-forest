# 开发与应用报告草稿

## 案例名称

我的计算森林：面向小学数学错因诊断与引导的教育智能体

## 一、开发背景

小学数学课堂中，教师每天都会看到大量“算错了”的结果，但真正耗时的不是判断对错，而是判断学生为什么错、下一步该怎样讲、怎样在有限时间内准备合适的巩固练习。

在真实教学中，主要有三类问题。第一，教师难以及时区分错因。同一道题的错误可能来自退位处理不稳定、进位遗漏、运算顺序错误、抄题转写问题或基础事实不熟，如果只看最终答案，很难快速决定该讲“方法”还是讲“习惯”。第二，短时、分层、教材对齐的反馈准备成本高。课堂小练通常只有 3 到 5 分钟，教师既要对齐当前教学进度，又要兼顾基础巩固和适度拓展。第三，学生在错题场景中常常只得到“错了”或“答案是什么”，缺少温和、可持续的学习反馈。

基于这些问题，我们提出“我的计算森林”。当前版本不再把“积累很多棵树”当作主功能，而是把重点放在：帮助教师快速看清错因、审核草案、组织短时反馈，并用温和成长语气保护孩子的学习节奏。

## 二、设计与开发

### （一）平台与技术选择

本项目采用 **Dify-first + FastAPI 工具服务** 的方式构建。

- `Dify` 负责教师侧工作流、知识库检索、提示词编排、节点日志和结果组织。
- `FastAPI` 负责结构化数学错因诊断、练习推荐与教师侧工具接口。
- `本地知识库` 负责教材方法、错因 taxonomy、讲评策略等材料。
- `教师审核` 作为统一边界，所有 AI 输出均默认待审核。

之所以这样拆分，是因为我们既希望发挥生成式 AI 在编排、表达和检索上的优势，又不能把数学对错完全交给大模型判断。

### （二）开发过程

开发过程中，我们借助生成式人工智能完成了需求梳理、提示词设计、Dify 工作流编排、接口联调、文档整理和材料打磨，并保留相关截图、日志和迭代记录，作为“AI 赋能开发”的证据。

当前已形成两层实现：

1. **规则诊断层**：通过 FastAPI 提供 `POST /api/diagnose`、`POST /api/practice/recommend`、`POST /api/dify/session-draft` 等接口，保证错因、证据和练习建议具备结构化输出。
2. **智能体编排层**：通过 Dify 工作流收集教师输入，调用本地接口，组织教师摘要、学生引导和短时练习草案，并保留可复盘的运行记录。

我们坚持一条硬边界：**算术正确性不由 LLM 判定**。大模型只负责整理、解释和组织输出，真正的结构化诊断由本地规则服务完成。

### （三）功能架构

当前成果的主要功能模块包括：

1. 错因诊断：根据题目、标准答案和学生作答输出主错因、证据和教师建议。
2. 分层引导：按标准/探索/挑战三种模式生成温和引导草案，但不直接给答案。
3. 短时练习：围绕当前错因生成 3 到 5 分钟内可完成的巩固题。
4. 教师审核：所有输出默认 `pending_teacher_review`，教师确认后再用于课堂。
5. 轻量成长语气：保留森林隐喻作为品牌表达和反馈语气，但不把它作为当前 MVP 主叙事。

## 三、应用过程与效果

本项目适用于小学数学课堂小练、作业讲评、错题订正和单元复习。

以当前演示案例为例，教师输入四年级题目 `402-178=`、标准答案 `224`、学生作答 `334`。系统不会只返回“错误”，而是给出主错因 `E03 退位错误`，并指出这道题涉及个位不够减、十位为 0 的连续借位处理。随后，系统继续生成教师摘要、学生引导和短时练习草案，供教师审核后使用。

这一流程带来的价值主要体现在三个方面：

1. **提高讲评效率**：教师不必先逐题人工归因，可以先查看结构化草案。
2. **增强针对性**：从“按题讲答案”转向“按错因讲方法”。
3. **保护学习节奏**：学生侧内容强调温和引导，不直接给答案，不强化排名和压力。

目前仓库中已经保留了 Dify 工作流、本地运行记录、接口示例、提示词材料、合成样例数据和文档说明，可支持后续整理为配套资源 ZIP。

## 四、创新与反思

本案例的创新主要体现在四点。第一，从“答对答错”推进到“错因 + 审核 + 引导”的教师工作流。第二，用 Dify 编排、本地规则诊断和知识库材料组合成可复盘的教育智能体。第三，坚持教师主导，所有结果默认待审核，保证课堂可用性与安全性。第四，保留森林这一品牌隐喻，但避免把课堂产品推向重游戏化。

当前也仍有不足：规则诊断对更复杂概念题和应用题的覆盖有限；低年级口算规则还需补强；多模态输入与长期成长系统仍属于后续扩展；教材版本适配和真实大规模应用证据仍需继续积累。

后续改进方向包括：补强 1-2 年级口算题型规则，完善教师审核反馈闭环，把“采纳、修改、驳回”反哺提示词和知识库，并在不偏离教师端主线的前提下继续评估成长语气配置的保留范围。

## 合规说明

本案例坚持教师主导、人机协同、数据脱敏和未成年人保护原则。演示材料统一使用合成样例数据，不展示真实学生姓名和正面画面。AI 生成的错因建议、引导话术和练习草案均需教师审核后使用；如材料中出现生成式人工智能生成的文本、图片或语音，将按要求标注“AI生成”。
