# 演示视频脚本

## 视频目标

用不超过 8 分钟的视频，按照《创AI》“案例概述 2 分钟以内 + 实现功能 5 分钟以内 + 应用情况 1 分钟以内”的结构，展示 **我的计算森林** 作为教育智能体的真实问题、实现方式与应用价值。

## 基本信息

- 项目名称：我的计算森林
- 副标题：教师主导的数学错因诊断与引导工作台
- 技术路线：Dify-first + FastAPI 工具服务
- 本地 Python 环境：`/home/lyzhang/miniconda3/envs/pyt0`
- 演示样例：四年级含 0 连续退位减法
- 视频时长：控制在 8 分钟以内，当前脚本建议约 5-6 分钟

## 录屏顺序

1. 用 20-30 秒讲清真实教学问题。
2. 打开 Dify 应用调试页或教师端原型页。
3. 输入题目、标准答案和学生作答，点击运行。
4. 切到 Dify Workflow 页面，展示节点链路和日志。
5. 展示诊断结果、教师摘要、学生引导和短时练习。
6. 展示教师审核边界、工作台状态或应用记录。
7. 用 30-60 秒总结应用效果与可复现材料。

## 旁白全文

```text
大家好，这个项目叫“我的计算森林”。

我们想做的，不是一个直接把答案塞给孩子的工具，而是一套由教师主导、能帮助老师快速讲清错因的小学数学教育智能体工作流。

老师只需要输入一道题、标准答案和学生作答，系统就会先帮助老师看清：孩子到底错在哪里。

这里展示的是一道四年级连续退位减法：402 减 178，学生写成了 334。

这类题真正难的地方，不只是结果对不对，而是个位不够减、十位又是 0 时，孩子有没有稳定处理借位关系。

我们的主流程搭在 Dify 上。Dify 负责收集输入、编排节点、记录日志；本地 FastAPI 工具负责给出结构化诊断和短时练习建议。

这里可以看到，系统给出的主错因是 E03，也就是退位错误，并给出了可复核的证据。

面向学生时，系统不会直接批评，也不会只丢出标准答案。它会先引导孩子自己想：哪一位不够减？借位后前一位有没有少 1？

然后，系统会给出 3 到 5 分钟以内的短练习，帮助孩子巩固同一类问题。

更重要的是，我们保留了森林这层温和的成长语气，但不把它做成喧宾夺主的收集玩法，避免课堂产品被游戏化带偏。

所有 AI 输出都默认待教师审核。这样既保留了生成式 AI 的效率，也保留了课堂应用所需要的准确性和安全边界。

这就是“我的计算森林”：用 Dify 做智能编排，用规则 API 保证诊断可靠，用教师审核和温和引导守住孩子的学习节奏。
```

## 拍摄注意事项

1. 严格按“案例概述 2 分钟以内 + 实现功能 5 分钟以内 + 应用情况 1 分钟以内”控制时长。
2. 不要只展示最终答案，要展示“错因、证据、引导、练习、审核边界”。
3. Dify 工作流画面要停留 5 秒以上，让评委看清节点设计。
4. 成长语气只展示“低压力、无排名、教师可控”的价值，不要拍成游戏化界面。
5. 全程使用脱敏样例数据，不出现真实学生信息。
6. 如视频中出现 AI 生成文本、图片或语音，需按指南标记“AI生成”。
