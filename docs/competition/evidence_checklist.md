# 《创AI》比赛证据留存清单

## 为什么要留证据

《创AI》开发与应用报告要求突出借助生成式人工智能进行开发的证据、应用过程与效果、完整代码和文档。项目从第一天开始就要保留可复现材料，并确保能对应到“教育智能体 + Dify 主入口 + 教师审核边界”这条主线。

## 开发过程证据

- 需求分析记录。
- 用户画像和真实教学痛点。
- 竞赛指南解读。
- Dify 平台选型与保留理由。
- 架构图和数据流图。
- API 设计文档。
- 数据模型文档。
- Git commit 记录。
- 使用 `pyt0` 环境的依赖和运行说明。
- 前端工作台与 Dify 的职责划分说明。

## AI 赋能开发证据

- 关键 Prompt 版本。
- Dify 工作流截图。
- LLM 辅助生成代码或文档的记录。
- 人工审核和修改记录。
- 生成内容标注方式。
- 知识库文档与工作流编排证据。

## 核心功能证据

- 错因标签体系。
- 诊断规则样例。
- 20-50 条合成作答样例。
- API 调用输入输出截图。
- 教师工作台截图。
- Dify draft run / workflow log 截图。
- 树种配置与鼓励语配置截图或 JSON 样例（仅作为可选成长语气证据）。
- 学生画像样例。
- 分层练习推荐样例。
- 班级错因 TopN 或热力图截图。
- 教师审核状态截图。

## 教学应用证据

- 教师试用反馈。
- 课堂或课后作业场景描述。
- 使用前后效率对比。
- 高频错因识别案例。
- 个性化练习案例。
- 学生订正效果案例。

## 合规证据

- 数据脱敏说明。
- 未成年人个人信息保护说明。
- 生成式 AI 使用边界说明。
- 教师审核机制说明。
- 演示数据来源说明。
- 不直接向学生提供答案的产品说明。
- “AI生成”标记规范说明。

## 演示视频素材

视频时长不超过 8 分钟，建议素材包括：

- 项目问题和场景介绍。
- Dify Agent / Workflow 交互片段。
- 自研 API 调用或页面演示。
- 错因诊断结果。
- 教师审核与工作台状态。
- 树种选择或鼓励语配置片段（可选，且不作为主镜头）。
- 学生画像和分层作业。
- 应用效果和后续计划。

## 配套资源 ZIP

建议最终 ZIP 包含：

- 完整代码或可复现开发流程。
- README。
- 安装文档。
- 使用手册。
- API 文档。
- Prompt 文档。
- Dify 工作流 DSL / 导入说明 / 运行截图。
- 演示数据。
- 测试记录。
- 主要截图。
- 合规说明与“AI生成”标记说明。

## 命名规范

建议所有证据文件按日期和主题命名：

```text
2026-05-01_dify_workflow_v1.png
2026-05-01_diagnose_api_sample.json
2026-05-01_teacher_feedback_01.md
```

这样后续写开发报告时可以快速还原迭代过程。
