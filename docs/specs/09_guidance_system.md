# 分层引导系统设计

> 对应第4次修改中"教材对齐·分层引导"部分。
> 本文档定义引导模式的数据模型和策略规则。

## 一、三种引导模式

| 模式 | 适用学生 | 引导内容 | 默认 |
| --- | --- | --- | --- |
| 标准模式 | 所有学生 | 只展示教材方法 | ✅ 默认 |
| 探索模式 | 基础良好、感兴趣 | 完成标准引导后，询问"想不想知道另一种算法？" | 可选 |
| 挑战模式 | 学有余力 | 直接问"除了课堂方法，你还能想到其他方法吗？" | 可选 |

### 判定规则

- 默认为标准模式
- 教师可手动设置学生的引导模式
- 系统可根据近期正确率自动建议（正确率 > 85% 连续两周 → 建议探索模式）
- 模式切换需要教师确认
- 无论哪种模式，进入学生侧前都需要教师审核

## 二、四步引导法（标准模式）

每道错题按以下四步引导，不直接给答案：

| 步骤 | 名称 | 说明 | 话术示例 |
| --- | --- | --- | --- |
| 1 | 接纳与安抚 | 不批评，先接纳 | "没关系，我们一起来看看这道题。" |
| 2 | 算理引导 | 使用教材方法引导 | "个位满10了，要捆成一捆，放到十位" |
| 3 | 总结方法 | 帮孩子提炼要点 | "两位数加法要注意什么？" |
| 4 | 巩固练习 | 推送同类题 | "来试试这题，用刚才的方法" |

### 数据模型

```json
{
  "guidance_session": {
    "diagnosis_id": "R0001",
    "mode": "standard",
    "steps": [
      {
        "step": 1,
        "type": "comfort",
        "message": "没关系，我们一起来看看这道题。"
      },
      {
        "step": 2,
        "type": "reasoning",
        "method": "textbook",
        "message": "个位满10了，要捆成一捆，放到十位",
        "visual_aid": "counter_method"
      },
      {
        "step": 3,
        "type": "summary",
        "message": "两位数加法要注意什么？"
      },
      {
        "step": 4,
        "type": "practice",
        "problems": ["236+178=", "457+286="]
      }
    ]
  }
}
```

## 三、按错因类型的引导策略

### MVP 需覆盖的策略

| 错因代码 | 教材方法 | 引导弹话术 | 教学工具 |
| --- | --- | --- | --- |
| E01 基础事实 | 加法累加/口诀表 | "3×4表示几个几相加？" | 口诀卡 |
| E02 进位错误 | 小棒法、计数器、竖式 | "个位满10了，要捆成一捆，放到十位" | 小棒/计数器 |
| E03 退位错误 | 破十法、借位点 | "个位不够减，从十位借一捆" | 数位表 |
| E04 数位对齐 | 方格纸竖式 | "写竖式时先对齐个位" | 方格纸 |
| E05 运算顺序 | 规则复述 | "先乘除，后加减" | 圈画标记 |
| E07 抄题转写 | 圈画法 | "先对照题目检查数字和符号" | 荧光笔 |
| E08 步骤遗漏 | 补步法 | "检查有没有只算了一部分" | 竖式模板 |
| E11 未验算 | 估算/逆运算 | "算完先问自己：这个答案合理吗？" | 估算表 |

## 四、拓展方法（探索/挑战模式）

### 原则

1. **教材方法优先：** 任何时候都先确保孩子掌握教材方法
2. **明确标注：** 拓展方法标注"拓展""你知道吗"，避免与教材方法混淆
3. **不强制：** 看到即可，不要求掌握
4. **正向引导：** "课堂方法很重要，这种方法可以帮你算得更快"

### 拓展方法示例（按错因）

| 错因代码 | 拓展方法 | 话术 |
| --- | --- | --- |
| E02 进位 | 凑十法 | "你知道吗？把一个数凑成整十，再加剩下的，会更快哦" |
| E03 退位 | 补数法 | "想不想知道：减法其实可以变成加法来做？" |
| E05 运算顺序 | 分配律简算 | "有时候换个顺序算，答案一样但更简单" |

### 数据模型

```json
{
  "extension_methods": {
    "E02": {
      "method_name": "凑十法",
      "mode": "exploration",
      "prerequisite": "已掌握标准进位方法",
      "dialogue": "你知道吗？把一个数凑成整十，再加剩下的，会更快哦"
    }
  }
}
```
