# 多模态输入设计

> 对应第4次修改中"多模态输入·技术适应孩子"部分。
> 本文档定义输入方式的数据模型和适配策略。MVP 阶段以键盘输入为主。

核心原则：**技术去适应孩子，而不是让孩子先学会复杂操作。**

## 一、三种输入方式

| 输入方式 | 适用年级 | 说明 | MVP 状态 |
| --- | --- | --- | --- |
| 🎤 语音输入 | 1-2年级 | 说出来就能答题 | 🔒 后续 |
| ⌨️ 键盘输入 | 3-6年级 | 数字键盘，快速准确 | ✅ 已实现 |
| 📷 拍照上传 | 所有年级 | 作业本练习，一拍即答 | 🔒 后续 |

## 二、年级适配策略

### 1-2年级（口算为主）

- 题型：20以内加减法、表内乘除法
- 预期输入：单数字答案
- 适配：
  - 语音输入（后续）：孩子说出答案，ASR 转文字后提取数字
  - 大按钮界面（后续）：触摸屏友好的数字选择
  - 键盘输入（已支持）：简单数字输入

### 3-4年级（竖式计算为主）

- 题型：三位数加减法、一位数乘两位数、简单混合运算
- 预期输入：多步竖式的最终答案 + 可选中间步骤
- 适配：
  - 键盘输入：数字 + 运算符
  - 步骤拍照（后续）：拍竖式过程

### 5-6年级（复杂运算为主）

- 题型：多位数乘除、四则混合、小数运算
- 预期输入：复杂答案 + 多步过程
- 适配：
  - 键盘输入：完整算式
  - 解题过程拍照（后续）：拍完整解题过程

## 三、输入数据模型

```json
{
  "input_mode": "keyboard",
  "source_metadata": {
    "input_method": "keyboard",
    "grade_adapted": true,
    "ocr_confidence": null,
    "asr_confidence": null
  }
}
```

当前 `AnswerRecord.source` 字段已预留此用途：
- `"manual"` — 手工/键盘录入（默认）
- `"voice"` — 语音输入（后续）
- `"photo"` — 拍照上传（后续）
- `"ocr"` — OCR 自动识别（后续）

## 四、MVP 中的体现

1. `schemas.py` 的 `AnswerRecord.source` 字段已支持扩展
2. 年级扩展到 1-2 年级后，诊断规则需增加口算题型的简单规则
3. 竞赛报告中可以描述完整的多模态输入愿景
4. 演示时可用合成数据展示不同输入方式的处理效果
5. Dify 首版主要承接教师侧工作流；学生侧多模态体验后续再产品化
