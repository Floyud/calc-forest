# 用户画像

## 一线小学数学教师

代表人物：三至六年级数学教师，通常同时负责一个或多个班级的授课、批改、讲评和家校反馈。

核心诉求：

- 快速知道班级共性错因。
- 找到每个学生反复出现的问题。
- 在双减要求下设计少量但有效的练习。
- 准备第二天课堂讲评和个别辅导。
- 不希望工具增加额外录入负担。

典型使用时刻：

- 批改课后作业后。
- 课堂练习结束后。
- 单元复习前。
- 家长沟通或学生面批前。

## 小学生

代表人物：小学三至六年级学生，正在建立数感、算理理解、竖式规范和验算习惯。

核心诉求：

- 知道自己错在哪里，而不是只看到红叉。
- 得到刚好能完成的练习，不被重复刷题压垮。
- 看到自己的进步。
- 在订正时获得提示，但不直接拿答案。

设计边界：

- 不直接开放生成式 AI。
- 不做公开排名。
- 不用连续打卡、强刺激奖励诱导沉迷。

## 教研组长 / 学校管理者

核心诉求：

- 看见作业提质减量的证据。
- 形成可复制的错因诊断和分层作业流程。
- 支持年轻教师快速掌握讲评重点。
- 在合规前提下推广到更多班级。

## 家长

核心诉求：

- 了解孩子具体薄弱点。
- 不再承担批改和讲解压力。
- 看到练习量可控、目标明确。

设计边界：

- 家长端不是首版重点。
- 不向家长输出过度标签化评价。
- 只展示教师确认后的阶段性反馈。
