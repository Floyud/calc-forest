# MVP 范围

## 首版目标

首版目标是用合成数据和手工录入数据跑通教育 Agent 闭环：

```text
作答记录 -> 错因诊断 -> 学生画像更新 -> 分层练习推荐 -> 教师审核摘要 -> 班级讲评建议
```

## 必做功能

1. **作答录入**
   - 教师录入题目、标准答案、学生答案。
   - 可选录入学生步骤、耗时、知识点。
   - OCR 只作为后续扩展，不阻塞首版。

2. **错因诊断**
   - 使用规则引擎判断基础计算错因。
   - 输出错因标签、证据、置信度和建议干预方式。
   - LLM 只生成教师可读解释，不改写诊断结论。

3. **学生画像**
   - 记录错因频次、知识点掌握度、近期趋势、订正效果。
   - 输出一句话画像摘要。

4. **分层练习**
   - A 层：基础补救。
   - B 层：标准巩固。
   - C 层：拓展变式。
   - 每份练习控制在 10 分钟以内，并标记待教师审核。

5. **教师看板摘要**
   - 班级高频错因 TopN。
   - 需要讲评的共性问题。
   - 需要个别关注的学生编号。

6. **教育 Agent**
   - Dify 工作流调用自研 API。
   - 输出教师可直接使用的诊断摘要和讲评建议。

## 延后功能

- 全自动拍照批改。
- 手写竖式步骤 OCR。
- 家长端。
- 真实学校多班级权限体系。
- 长周期学习预测模型。
- 移动端小程序。

## 演示数据范围

首版准备 20-50 条合成样例，优先覆盖整数计算场景。每类题型至少准备 3 个错误样例，并覆盖 5 个匿名学生编号。

### 首版题型

1. 三位数加减法。
   - 重点错因：`E02` 进位错误、`E03` 退位错误、`E04` 数位对齐错误、`E11` 未验算。
2. 含 0 的连续退位减法。
   - 重点错因：`E03` 退位错误、`E07` 抄题/转写错误、`E08` 步骤遗漏、`E11` 未验算。
3. 一位数乘两位数。
   - 重点错因：`E01` 基础事实错误、`E02` 进位错误、`E04` 数位对齐错误。
4. 简单四则混合运算。
   - 重点错因：`E05` 运算顺序错误、`E01` 基础事实错误、`E08` 步骤遗漏。

### 首版覆盖标签

- 必须覆盖：`E01`、`E02`、`E03`、`E04`、`E05`、`E07`、`E08`、`E11`。
- 可选覆盖：`E09`，仅在学生明显套错方法时使用。
- 暂不作为首版演示重点：`E06` 小数点/分数单位错误、`E10` 审题与单位理解错误。

所有样例使用合成学生编号，如 `S001`、`S002`，不使用真实姓名。

## 首版验收

- 每条错题至少输出 1 个错因标签。
- 每个错因都能看到证据。
- 每个学生能生成画像摘要。
- 每份推荐练习有层级、题量、预计耗时和推荐理由。
- 教师端输出不直接发布给学生，状态为待审核。
