# 产品愿景：小数诊疗舱

## 一句话定位

**小数诊疗舱** 是面向小学数学教师的计算错因诊断与分层练习教育 Agent，帮助教师把“批改后的对错结果”转化为“可行动的教学诊断”。

## 真实问题

小学数学计算错误高频出现，但很多错误被笼统归因为“粗心”。在真实课堂里，教师更需要知道：

- 学生到底是不会算，还是步骤习惯有问题。
- 一个学生是否反复出现同类错因。
- 全班下一节课最该讲哪个共性问题。
- 双减要求下，如何少布置但布置得更准。

## 目标用户

首要用户是小学三至六年级数学教师。教师在批改课后作业、课堂练习、单元复习时使用 Agent 完成诊断、讲评和分层练习设计。

学生是间接受益者。学生不直接使用生成式 AI 获取答案，只接收教师审核后的练习和成长反馈。

## 核心价值

- **从判对错到诊断错因**：每道错题输出错因标签、证据和订正建议。
- **从统一刷题到精准补救**：根据错因和学生画像生成少量、有针对性的练习。
- **从个人经验到班级看板**：把教师批改中的隐性判断沉淀为班级错因热力图和讲评建议。
- **从技术替代到教师增能**：AI 只做辅助诊断和草案生成，最终由教师审核。
- **从炫技到合规落地**：默认脱敏、教师主导、生成内容标注、学生端不直接开放生成式 AI。

## 比赛叙事

《创AI》关注教师借助生成式人工智能自主开发智能教学工具。本项目的叙事重点是：

> 一线小学数学教师面对计算错误反复发生和分层作业设计成本高的问题，借助 Dify/Coze 等智能体平台完成教学 Agent 的工作流编排，并自主开发数学错因诊断服务，形成“诊断、画像、推荐、讲评、反馈”的闭环。

## 首版成功样子

第一版不追求全自动拍照批改，而是先跑通可信闭环：

1. 教师录入或校正一条学生作答。
2. 系统输出错因标签、诊断证据和订正建议。
3. 系统更新学生画像。
4. 系统生成 10 分钟以内的分层练习草案。
5. 教师看到班级高频错因和课堂讲评建议。

## 不做什么

- 不做拍照搜答案工具。
- 不让小学生独立使用生成式 AI 聊天。
- 不用 LLM 直接决定数学对错。
- 不采集非必要个人信息。
- 不设计排名刺激和沉迷式游戏机制。
