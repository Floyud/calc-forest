# 诊断引擎决策逻辑（教师参考）

> 本文件描述系统诊断引擎的判定流程，帮助教师理解"为什么系统会给出这个错因标签"。错因代码的详细定义见 [00_shared/error_codes.md](../00_shared/error_codes.md)。

---

## 诊断输入

系统收到一组数据后开始诊断：

| 输入项 | 说明 | 必填 |
|---|---|---|
| 题目 | 原始算式 | 是 |
| 正确答案 | 标准答案 | 是 |
| 学生答案 | 学生提交的答案 | 是 |
| 学生步骤 | 学生的计算过程（如有） | 否 |
| 年级 | 学生年级 | 否 |

---

## 诊断决策树

当系统收到一组 (题目, 正确答案, 学生答案) 时，按以下顺序依次检测。一旦匹配某条规则且置信度达标，就标记对应错因并停止检测。如果所有规则都不匹配，标记 E99。

### 第1步：答案完全一致

比较学生答案与正确答案（去除空格、统一格式后）。

- **一致** → 正确，无需诊断，流程结束
- **不一致** → 进入第2步

### 第2步：检查抄题/转写错误（E07）

将学生列式或步骤中的数字与原题数字逐位比较。

**判定条件**：
- 原题中的某个数字在学生步骤中变成了另一个数字（如 125→152、306→360）
- 原题中的运算符号在学生步骤中变了（如 +→-、×→÷）
- 用学生步骤中"抄错"的数字重新计算，能得到学生的答案

**匹配** → E07 抄题/转写错误
**不匹配** → 进入第3步

### 第3步：检查括号处理（E05 部分）

原题中是否有括号，且学生答案符合"忽略括号后的计算结果"。

**判定条件**：
- 原题含括号，去掉括号后按从左到右顺序算出的结果 = 学生答案

**匹配** → E05 运算顺序错误
**不匹配** → 进入第4步

### 第4步：检查运算顺序（E05）

原题是否为混合运算（含两种及以上运算类型）。

**判定条件**：
- 按从左到右顺序（而非先乘除后加减）计算原题，结果 = 学生答案
- 或者：学生跳过了中间的某步运算

**匹配** → E05 运算顺序错误
**不匹配** → 进入第5步

### 第5步：检查退位错误（E03）

原题是否包含减法。

**判定条件**：
- 减法运算中，某一位"不够减"时学生直接用了"大减小"（如 2-8 写成 8-2=6）
- 学生答案与正确答案的差 = 10 的倍数（暗示某位没有退位）
- 被减数中间有0的场景优先检查

**匹配** → E03 退位错误
**不匹配** → 进入第6步

### 第6步：检查进位错误（E02）

原题是否包含加法或乘法。

**判定条件**：
- 加法竖式中，某一位的结果恰好比正确值小1（漏加进位）
- 或某一位比正确值大10（多进了位）
- 乘法竖式中，部分积相加时进位未加

**匹配** → E02 进位错误
**不匹配** → 进入第7步

### 第7步：检查数位/小数位对齐（E04）

**判定条件**：
- 学生答案与正确答案的差 = 10^n（n为整数），暗示数位整体偏移
- 小数加减中，学生按"末位对齐"计算的结果 = 学生答案
- 小数乘法中，小数位数与正确答案不符

**匹配** → E04 数位对齐错误
**不匹配** → 进入第8步

### 第8步：检查步骤遗漏（E08）

原题是否为多步运算（脱式、简便运算、图形公式计算等）。

**判定条件**：
- 学生只完成了前几步就停止，跳过了后续步骤
- 圆锥体积 = 圆柱体积（漏×1/3的典型信号）
- 圆柱表面积 = 侧面积（漏+底面积）
- 利息计算 = 本金×利率（漏×存期）

**匹配** → E08 步骤遗漏
**不匹配** → 进入第9步

### 第9步：检查小数/分数单位（E06）

**判定条件**：
- 分数加减时，学生把分子加分子、分母加分母得到的结果 = 学生答案
- 小数乘法中，小数点位置与正确答案偏移了若干位
- 百分数化小数出错（如 4%→0.4 而非 0.04）

**匹配** → E06 小数点/分数单位错误
**不匹配** → 进入第10步

### 第10步：检查基础事实（E01）

**判定条件**：
- 学生的计算过程可还原为某个中间步骤的基础运算出错
- 错误答案与"某步基础事实出错后的计算结果"吻合
- 差值为1-3以内，且无规律性偏移

**匹配** → E01 基础事实错误
**不匹配** → 进入第11步

### 第11步：检查算理理解（E09）

**判定条件**：
- 学生使用了错误的公式或方法（如 r²→r×2）
- 方法选择与题目不匹配
- 迁移失败：类似题目能做对，变式题出错

**匹配** → E09 算理理解不足
**不匹配** → 进入第12步

### 第12步：检查审题/单位（E10）

**判定条件**：
- 学生答案 = 正确计算了"另一个问题"的结果（答非所问）
- 单位换算方向搞反（km→cm时除以而非乘以）
- 题意理解错误（"求剩下的"算成"求用去的"）

**匹配** → E10 审题与单位理解错误
**不匹配** → 进入第13步

### 第13步：检查未验算（E11）

**判定条件**：
- 学生答案明显不合理（量级错误、符号错误、结果为负但应为正等）
- 但上述所有规则均未匹配
- 可能是多种错误的复合结果

**匹配** → E11 习惯性未验算
**不匹配** → E99 暂未识别

## 多重错误处理

同一道题可能同时存在多种错误类型。系统的处理原则：

1. **优先标记最关键的错误**：按决策树从上到下的顺序，第一个匹配的错因被标记为主错因
2. **可选标记次错因**：如果置信度允许，系统会标注第二个可能的错因
3. **教师可修正**：所有诊断结果标记为 `pending_teacher_review`，教师可以修改或补充

---

## 教师审核要点

系统诊断完成后，教师需要关注以下几点：

1. **判断是否合理**：系统给出的错因是否与学生的实际计算过程一致
2. **补充步骤信息**：如果学生只提交了答案没有过程，E99 的概率会升高。要求学生补写过程后重新诊断
3. **人工修正**：系统可能误判，教师可以根据经验修改错因标签
4. **关注模式**：如果同一学生反复出现同一种错误，说明该类型需要专项干预
